# LAST SALVAGE — 포트폴리오 정리

> 모바일 픽셀 로그라이크. **기획서에 약속만 있고 비어 있던 시스템들을, 모바일 60fps 예산을
> 깨지 않는 데이터 주도 아키텍처로 실제 구현해 채워 넣은** 프로젝트.

## 2. 기술 스택 & 아키텍처 하이라이트

- **씬 구성**: Preload / Combat / Hub / Settings. 전투는 허브 조작 중에도 멈추지 않는 **상시 자동 진행**.
- **에셋 지연 로드**: 적·보스·지역 배경을 필요한 시점에 1장씩 로드(선행 풀로드 없음) → 모바일 초기 로딩 최소화.
- **데이터 주도 설계**: 무기 메카닉(`crafting.js`), 적 행동(`enemyBehaviors.js`), 지역 효과(`regions.js`)를
  **타입드 config + 디스패치 테이블**로 분리 → 콘텐츠 추가 시 데이터 한 줄로 확장.
- **모바일 60fps 규율 (코드 전반에 일관)**:
  - per-enemy Phaser 타이머 **0개** — 모든 상태(감전·DoT·속박·독웅덩이·콤보·보스 페이즈)를
    `scene.time.now` 비교 + 단일 update 루프로 틱.
  - tint 9단계 **사전계산 팔레트** + 동일 인덱스 setTint 생략으로 GPU 상태변경 최소화.
  - 데미지 숫자/VFX **오브젝트 풀링**(`_dmgPool`, 상한 24), 매 프레임 임시배열(filter/sort) 회피.
  - 지속 이펙트는 **draw-once Shape + tween**(매 프레임 `graphics.clear()` 재그리기 금지).

## 4. 이번 라운드 핵심 기여

### A. 전투 메카닉 (긴장·내러티브)
- **보스 분노 페이즈** — HP 50% 이하에서 속도 ×1.9·공속 35%↑ + ENRAGED 연출. "버티다 분노하면?"의 분기점.
- **LAST SALVAGE 사투 모드** — HP 25% 이하에서 공격력 +40%. 게임 타이틀과 직결된 "죽기 직전 역전" 장치.
- **무피해 클린스윕 보너스** — 웨이브 무피해 클리어 시 희귀 재료 보너스(능동적 빌드 동기).

### B. 적 행동 차별화 — 기획서의 "약속" 구현
기존엔 모든 적이 "직진→근접타" 단일 AI였음. 기획서 5종 패턴을 **새 AI 프레임워크 없이**
이미 존재하던 4개 seam(이동·피격·접촉·사망)에 한 줄 훅으로 데이터 주도 구현:
- **정면방어(guard)**, **사망폭발(explode)**, **측면대시(flank)**, **속박(grab + 테더)**, **독웅덩이(지속 존)**
- **엘리트 적**(주입형 hpMult·tint·스케일), **보스 페이즈**(콜로서스 방어업 / 헤럴드 좀비 소환)
- 신설 `src/constants/enemyBehaviors.js` — 각 행동이 자기 훅만 선언, scene/Phaser 비참조, 부작용은 ctx 주입.

### C. 전략 레이어
- **지역 패시브 전투버프** — 8개 지역에 속성 배율(독성늪 TOXIC +40% 등). "지형을 읽고 무기를 준비"하는 사전 계획 유발.
- **킬 콤보 멀티플라이어** — 연속 킬 ×10에서 드롭 등급↑. 자동전투의 흐름을 시각화.
- **합성 힌트 하이라이트** — 제작 가능 레시피 금색 글로우, 1개 부족은 옅은 강조(온보딩 + 베테랑 의사결정).

### D. 가독성·완성도
- 적 상태 색점(statusPip), HUD 세로 리듬 정돈, 설정 터치타겟 확대, 위협 적 등장 경고 라벨.

### 규모
9개 파일 **+831 / −51**, 신규 `enemyBehaviors.js`. 3개 Phase 모두 빌드 통과 + 자체 perf 감사에서 발견된 누수 3건 수정.

## 5. 에셋 파이프라인 — 캐릭터 일관성 문제 (★ 차별화)

플레이어는 **동일 인물·8단계 의상 진화**. 여기에 walk/attack/hit/death 애니메이션을 입히려면
"얼굴·정체성은 고정, 옷은 단계별, 포즈만 변형"이라는 까다로운 일관성 문제를 풀어야 했다.

**해법 — 포즈 골격을 단계와 분리:**
1. **ControlNet OpenPose 골격 12프레임을 코드로 deterministic 생성**(`tools/generate_pose_skeletons.py`,
   정면 3/4·COCO-18 표준). 8단계 의상 전부에 **같은 골격**을 적용 → 옷이 달라도 애니 타이밍이 정확히 일치.
2. **생성 가이드 문서화**(`PLAYER_ANIMATION_GUIDE.md`) — "기존 stage PNG를 레퍼런스로 넣고 포즈만 바꾸는
   image-to-image"라는 원칙 + 고정 절/부정 프롬프트/네이밍 규칙으로 정체성 드리프트 차단.
3. **정규화 파이프라인**(`scripts/normalize-anim-frames.mjs`) — 생성기가 캔버스/스케일을 제각각 뱉어도
   알파 bbox 실측 → 단계 기준(서있는 프레임)으로 키 통일 → **발끝을 고정 베이스라인에 정렬** → 동일 캔버스로 패딩.
   검증 결과 발끝 흔들림 **Δ0.000**(원본은 0.86~0.92로 들쭉날쭉했음).

> 포인트: "AI 생성 + 결정론적 정렬 코드"를 결합해, 손으로 다듬지 않고도 게임에 바로 쓸 수 있는
> 일관된 스프라이트를 뽑는 재현 가능한 파이프라인을 만들었다.

## 6. 작업 방식 — AI 에이전트 팀 운영

단일 어시스턴트가 아니라 **역할별 6개 서브에이전트 팀**(team-lead / designer / developer /
motion-engineer / perf-engineer / ideator)을 구성·운영. 의미 있는 작업은:
team-lead가 격차 분석·분담 → 역할별 병렬 작업 → **perf·접근성·일관성 게이트** 통과 → team-lead 종료 보고.
이번 라운드도 ideator(아이디어)·developer(구현)·designer(가독성)·perf-engineer(성능 감사)가 각 단계에 관여.

> 단순 코딩이 아니라 **"역할 분리 + 품질 게이트 + 학습 누적(team-memory)"의 워크플로를 설계해 운영**한 경험.

## 7. 내가 특히 신경 쓴 부분

- **기획서 = 약속, 코드 = 이행.** 화려한 신규 기능보다, 기획서에 적어두고 비워뒀던 시스템(적 행동·보스 페이즈·
  지역 효과)을 **실제로 구현해 격차를 메우는 것**을 우선했다.
- **모바일 60fps는 타협 불가.** 새 기능을 넣을 때마다 "타이머 0 / draw-once / 풀 재사용" 규칙을 지켰는지
  **별도 perf 감사**로 검증하고, 발견된 누수는 그 자리에서 수정.
- **작은 화면 가독성.** 새 위험 요소(속박·독웅덩이·엘리트)는 반드시 색점·테더·경고 라벨로 "분명히 읽히게".
- **모션은 의미 있을 때만 + `prefers-reduced-motion` 존중.** 콤보·경고·글로우·분노 플래시 전부 모션 폴백 제공.
- **캐릭터 정체성 보존에 집착.** 애니메이션을 위해 캐릭터 얼굴·의상이 1px도 변하지 않도록 파이프라인 자체를 설계.
- **유지보수성 + 판을 흔드는 똑똑한 해법.** 적 AI를 새 프레임워크 없이 기존 seam에 데이터 한 줄로 끼우는 식.
