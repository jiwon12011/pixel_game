# 미사용 애셋/헬퍼는 기획서 힌트다

**패턴**: 이 레포(LAST SALVAGE)에서 "재미/기능 보완" 요청이 오면, 신규 시스템을 설계하기 전에 **이미 로드되지만 미연결인 재료**부터 찾는다.

**근거 (2026-07 디자인·사용성·재미 보완 라운드)**:
- `NOTIFICATION_BADGE` 텍스처가 매니페스트에 로드돼 있었지만 사용처 0 → 기획서의 "탭 픽셀 뱃지" 미구현 신호였다. 뱃지 시스템으로 연결해 핵심 루프 신호를 복구.
- `CombatDirector.enemiesInRange` 헬퍼(관통용)가 이미 있었다 → 설치형 sweep(다중 타격)을 신규 오브젝트 0개로 구현하는 이음새가 됐다.
- `player.getAttackCooldown()` 인터페이스 패턴 → 옆에 `getAttackRange()/getTargetMode()`를 나란히 추가하는 것만으로 무기 계열(사거리/타겟팅) 차별화가 성립.

**적용법**: GAME_DESIGN.md 약속 ↔ 코드 실물 사이의 갭 목록을 먼저 만들고, 갭마다 "이미 있는 이음새(미사용 텍스처·헬퍼·인터페이스)"를 찾아 최소 변경으로 잇는다. 기존 파이프라인 위 숫자 몇 개 추가가 신규 시스템보다 항상 먼저다.
